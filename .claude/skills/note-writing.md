# Rally 노트 작성 방법론

Rally 워크스페이스에 노트를 생성할 때 따르는 원칙과 패턴.

## 언제 적용하는가

- `mcp__rally__write_content`로 노트(type: "note")를 생성할 때
- 기존 노트를 업데이트할 때

## 노트 구조 원칙

### 1. 제목은 내용을 즉시 파악할 수 있게

```
좋음: "터미널 패널 구현 개요"
나쁨: "터미널"
좋음: "v1.2.0 캔버스 기능 추가 정의"
나쁨: "캔버스 메모"
```

여러 노트를 만들 때는 번호를 붙여 순서를 명확히 한다:
- "1. 개요", "2. Store 설계", "3. UI 구현"

### 2. 첫 문단은 한줄 요약

노트 시작 부분에 이 노트가 무엇인지 한줄로 요약한다.
상세 내용은 그 아래에 배치한다.

```markdown
# 터미널 패널 Store

> `features/terminal/terminal-panel-store.ts` — 터미널 패널의 열림/닫힘 상태를 관리하는 Zustand 스토어

## 상태 구조
...
```

### 3. 계층 구조는 2단계까지

`##`과 `###`만 사용한다. `####` 이하는 가독성이 떨어진다.
내용이 깊어지면 별도 노트로 분리한다.

### 4. 코드 블록에는 언어 태그

````markdown
좋음:
```typescript
const store = create<State>((set) => ({ ... }))
```

나쁨:
```
const store = create((set) => ({ ... }))
```
````

### 5. 표 데이터는 마크다운 표(| | |) 대신 `mcp__rally__write_content`의 `type: "table"`을 사용

마크다운 표(| | |) 대신 `mcp__rally__write_content`의 `type: "table"`을 사용하여 CSV로 생성한다.
Rally 앱에서 표를 인터랙티브하게 볼 수 있다.
CSV를 따로 생성했다면 관련 노트에 `linkItems`로 연결하여 맥락을 유지한다.

## 노트 분리 기준

하나의 주제를 여러 노트로 나눌 때:

- **개요 노트 1개**: 전체 구조, 관계, 핵심 포인트
- **세부 노트 N개**: 레이어/모듈/단계별로 분리
- **적정 개수**: 3~10개 (1~2개면 분리 불필요, 10개 넘으면 그룹화 고려)

분리 기준:
- 각 노트가 독립적으로 읽혀야 한다
- 하나의 노트가 스크롤 없이 읽기 힘들면 분리한다
- "이 부분만 다시 보고 싶다"는 단위로 나눈다

## 폴더 배치

- `mcp__rally__list_items`로 기존 폴더 구조를 먼저 확인한다
- 기존에 맞는 폴더가 있으면 그곳에 넣는다
- 없으면 `mcp__rally__manage_folders`로 생성한다
- 폴더명은 주제를 명확히 나타내는 짧은 이름으로 한다
