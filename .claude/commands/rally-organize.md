# Rally 워크스페이스 정리

워크스페이스의 항목을 분석하고 정리합니다.

## 입력

$ARGUMENTS

## 실행 규칙

### 1단계: 현황 분석

1. `mcp__rally__list_items`로 전체 폴더/노트/테이블/캔버스 파악
2. `mcp__rally__list_todos`로 할일 현황 파악 (filter: all로 완료 포함)

분석 항목:

- 폴더 구조 및 항목 분포
- 빈 폴더와 그 폴더에 어울리는 항목이 다른 곳에 있는지
- 폴더에 속하지 않은 루트 항목
- 성격이 비슷한 항목이 흩어져 있는지
- 할일 상태 분포 (할일/진행중/완료/보류 건수)

### 2단계: 지시에 따라 분기

#### 입력이 없거나 "현황", "분석"인 경우

분석 결과만 리포트로 출력한다:

```
## 워크스페이스 현황

### 구조
- 폴더: N개 (빈 폴더: N개)
- 노트: N개 / 테이블: N개 / 캔버스: N개
- 루트 항목: N개

### 할일
- 활성: N건 (할일 N / 진행중 N / 보류 N)
- 완료: N건

### 정리 제안
- (구체적 제안 목록)
```

#### 입력이 구체적 정리 지시인 경우

해당 작업을 수행한다.

### 정리 원칙 (우선순위 순)

1. **재배치 우선**: 삭제보다 적절한 위치로 이동을 먼저 고려한다
   - 빈 폴더가 있으면 → 그 폴더에 어울리는 항목을 찾아 이동
   - 루트 항목이 있으면 → 성격에 맞는 기존 폴더로 이동
   - 성격이 비슷한 항목이 흩어져 있으면 → 한 폴더로 모은다
2. **구조 개선**: 폴더명 변경, 계층 재구성으로 일관성 확보
3. **삭제는 최후 수단**: 명백히 불필요한 경우에만 제안하며, 반드시 확인 후 실행

### 가능한 작업

**항목 재배치**

- `mcp__rally__manage_items`로 노트/테이블을 성격에 맞는 폴더로 이동
- 제목, 내용, 폴더명을 기준으로 가장 적절한 위치를 판단

**폴더 정리**

- `mcp__rally__manage_folders`로 폴더 생성/이동/이름변경
- 빈 폴더에 어울리는 항목을 채우거나, 정말 불필요하면 삭제 제안

**할일 정리**

- `mcp__rally__manage_todos`로 할일 상태 변경
- 보류 할일 리뷰 및 상태 업데이트

### 안전 규칙

1. **삭제는 반드시 대상 목록을 먼저 보여주고 사용자 확인 후 실행한다**
2. 이동/이름변경은 즉시 실행 가능
3. 삭제보다 이동을 권장한다 — "이 항목은 X 폴더에 넣는 게 좋겠습니다"
4. 한 번에 너무 많은 변경을 하지 않는다 (10건 이상이면 분할 제안)

## 출력 형식

작업 완료 후 변경 사항을 요약:

- 수행한 작업 (이동/이름변경/생성/삭제)
- 변경된 항목 목록
- 정리 후 폴더 구조 트리
