# Rally 맥락 수집

워크스페이스에서 주제와 관련된 자료를 탐색하고 종합 요약합니다.

## 입력

$ARGUMENTS

## 실행 규칙

### 1단계: 검색 범위 결정

입력을 분석하여 검색 전략을 결정한다:

- 키워드/주제명이면 → `mcp__rally__search_notes`로 노트 검색 + `mcp__rally__list_items`로 폴더 구조 탐색
- 특정 폴더/기능이면 → `mcp__rally__list_items`로 해당 폴더 내 항목 파악
- 할일 참조면 → `mcp__rally__list_todos`로 할일 + linkedItems 확인

### 2단계: 자료 수집

검색 결과에서 관련 항목을 깊이 읽는다:

1. `mcp__rally__search_notes`로 키워드 매칭 노트를 찾는다
2. `mcp__rally__list_items`로 관련 폴더 내 노트/테이블/캔버스를 파악한다
3. 발견된 항목을 `mcp__rally__read_content` (노트/테이블) 또는 `mcp__rally__read_canvas` (캔버스)로 상세 읽기한다
4. `mcp__rally__list_todos`로 관련 할일과 그 linkedItems도 확인한다

수집 가이드:

- 제목이나 폴더 경로에 키워드가 포함된 항목을 우선 수집
- 할일의 linkedItems를 따라가며 연관 자료를 확장 탐색
- 캔버스는 노드 구조와 연결 관계를 파악하여 전체 그림을 이해

### 3단계: 종합 요약

수집된 자료를 다음 구조로 정리하여 출력한다:

1. **한줄 요약**: 이 주제가 무엇인지
2. **관련 자료 목록**: 찾은 노트/테이블/캔버스/할일 (제목 + 위치)
3. **핵심 내용 정리**: 자료에서 추출한 주요 내용을 주제별로 구조화
4. **할일 현황**: 관련 할일이 있으면 상태와 함께 정리
5. **참고 사항**: 누락되었을 수 있는 부분, 추가 확인이 필요한 부분

## 출력 형식

```
## [주제명] 맥락 요약

### 관련 자료
- 📄 노트: [제목] (폴더/경로)
- 📊 테이블: [제목] (폴더/경로)
- 🗺️ 캔버스: [제목]
- ✅ 할일: [제목] (상태)

### 핵심 내용
(주제별 구조화된 요약)

### 할일 현황
(관련 할일 상태 정리)
```
