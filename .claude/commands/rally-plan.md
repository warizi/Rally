# Rally 기획 & 할일 분해

아이디어나 요구사항을 받아 기획 노트를 작성하고, 실행 할일로 분해하여 캔버스로 시각화합니다.

## 입력

$ARGUMENTS

## 참조 Skills

노트 작성 시 `.claude/skills/note-writing.md`의 원칙을 따른다.
할일 분해 시 `.claude/skills/todo-management.md`의 원칙을 따른다.
캔버스 생성 시 `.claude/skills/canvas-design.md`의 원칙을 따른다.

## 실행 규칙

### 1단계: 요구사항 분석

1. 입력에서 기능명과 요구사항을 파악한다
2. `mcp__rally__search_notes`로 기존 관련 자료가 있는지 확인한다
3. `mcp__rally__list_todos`로 이미 등록된 관련 할일이 있는지 확인한다
4. 기존 자료가 있으면 `mcp__rally__read_content`로 읽어 참고한다

### 2단계: 기획 노트 작성

1. `mcp__rally__list_items`로 기존 폴더 구조를 확인한다
2. 적절한 폴더가 없으면 `mcp__rally__manage_folders`로 생성한다 (보통 `기능/` 하위)
3. `mcp__rally__write_content`로 기획 노트를 작성한다

기획 노트 구성:

- **기능 정의**: 무엇을, 왜 만드는지
- **범위**: 포함/제외 사항
- **기술 고려사항**: 사용할 기술, 아키텍처 영향, 주의점
- **UI 흐름** (해당 시): 사용자 인터랙션 순서

### 3단계: 할일 생성

`mcp__rally__manage_todos`로 할일을 생성한다:

1. **상위 할일 1개**: 기능명을 제목으로 생성
   - 기획 노트를 `linkItems`로 연결
   - 적절한 priority 설정
2. **하위 할일 N개**: 실행 단위로 분해하여 `subtodos`로 생성
   - 각 하위 할일은 독립적으로 완료 가능한 단위
   - 의존 순서가 있으면 제목에 번호를 붙인다

분해 가이드:

- 한 하위 할일 = 하나의 작업 세션에서 끝낼 수 있는 크기
- DB 변경, 서비스 로직, UI 구현, 연동 등 레이어별로 분리
- 너무 세분화하지 않는다 (3~7개가 적당)

### 4단계: 캔버스 시각화

`mcp__rally__create_canvas`로 기획 구조를 시각화한다:

- 기획 노트를 `note` 타입 노드로 배치 (refId 사용, 최소 width: 400, height: 300)
- 각 하위 할일을 `text` 타입 노드로 배치
- 의존 관계나 실행 순서를 `edges`로 연결
- 색상으로 구분: 기획(보라), 할일(파랑), 주의사항(빨강)

## 출력 형식

작업 완료 후 생성된 항목을 요약:

- 기획 노트 (제목 + 폴더)
- 할일 트리 (상위 + 하위 목록)
- 캔버스명 + 노드/엣지 수
