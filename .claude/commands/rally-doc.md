# Rally 문서화 & 시각화

주어진 대상을 분석하여 Rally 워크스페이스에 정리 노트를 만들고 캔버스로 시각화합니다.

## 입력

$ARGUMENTS

## 실행 규칙

### 1단계: 대상 파악

- 입력이 코드 경로/패턴이면 → 해당 소스코드를 읽고 구조를 분석
- 입력이 개념/주제이면 → 프로젝트 내 관련 코드와 문서를 탐색하여 정리
- 입력이 기존 Rally 노트 참조이면 → 해당 노트를 읽어서 재구성

### 2단계: 노트 생성

1. `mcp__rally__manage_folders`로 주제 이름의 폴더를 생성한다
2. 분석 내용을 논리적 단위로 분리하여 각각 `mcp__rally__write_content`로 노트를 생성한다
3. 노트 구성 가이드:
   - 첫 번째 노트는 항상 **전체 개요** (구조, 관계, 핵심 포인트)
   - 이후 노트는 세부 주제별로 분리 (3~10개가 적당, 내용이 많으면 더 작성)
   - 각 노트에 코드 블록, 다이어그램(텍스트) 등 적극 활용
   - 노트 제목에 번호를 붙여 순서를 명확히 한다 (예: "1. 개요", "2. 상세")
   - **테이블/표 데이터가 필요한 경우** `mcp__rally__write_content`의 `type: "table"`을 사용하여 CSV 형식으로 생성한다 (마크다운 표 대신 table 타입 활용)

### 3단계: 캔버스 시각화

1. `mcp__rally__create_canvas`로 캔버스를 생성한다
2. 캔버스 구성 가이드:
   - 생성한 노트를 `note` 타입 노드로, 테이블을 `csv` 타입 노드로 참조 (refId 사용)
   - `note`, `csv` 등 참조 노드는 내용이 잘 보이도록 충분히 크게 설정한다 (최소 width: 400, height: 300)
   - 핵심 개념/계층/흐름을 `text` 타입 노드로 추가
   - 관계와 흐름을 `edges`로 연결
   - 색상으로 계층/카테고리를 구분
   - 위에서 아래로 흐르는 레이아웃 기본, 필요시 좌→우 흐름 추가

## 출력 형식

작업 완료 후 생성된 항목을 요약:
- 폴더명
- 노트 목록 (번호 + 제목 + 한줄 설명)
- 캔버스명 + 노드/엣지 수
