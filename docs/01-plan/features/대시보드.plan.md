# Plan: 대시보드

## 개요

워크스페이스의 모든 데이터를 한눈에 볼 수 있는 종합 대시보드.
할 일 통계, 오늘 일정, 최근 노트/캔버스, 파일 현황 등을 위젯 카드 형태로 집계하여 표시한다.

## 배경

### 현재 상태 분석

- `DashboardPage`가 stub으로 존재 (`src/renderer/src/pages/dashboard/ui/DashboardPage.tsx`)
- 사이드바에 대시보드 메뉴 이미 등록됨 (앱 시작 시 기본 탭)
- `pane-routes.tsx`에서 `/dashboard` → `DashboardPage` lazy import 완료
- 모든 엔티티의 조회 API와 React Query 훅이 이미 구현되어 있음
- **추가 백엔드 작업 없이** 기존 API만으로 대시보드 구성 가능

### 사용 가능한 기존 API/Hooks

| 데이터 | API / Hook | 비고 |
|--------|-----------|------|
| 할 일 | `useTodosByWorkspace(workspaceId, { filter })` | all/active/completed 필터 |
| 일정 | `useSchedulesByWorkspace(workspaceId, { start, end })` | 날짜 범위 필터 |
| 일정 (전체) | `useAllSchedulesByWorkspace(workspaceId)` | 전체 조회 |
| 노트 | `useNotesByWorkspace(workspaceId)` | preview, updatedAt 포함 |
| 캔버스 | `useCanvasesByWorkspace(workspaceId)` | updatedAt 포함 |
| CSV | `window.api.csv.readByWorkspace(workspaceId)` | React Query hook 필요 시 작성 |
| PDF | `window.api.pdf.readByWorkspace(workspaceId)` | React Query hook 필요 시 작성 |
| 이미지 | `window.api.image.readByWorkspace(workspaceId)` | React Query hook 필요 시 작성 |
| 태그 | `window.api.tag.getAll(workspaceId)` | 태그 목록 |

## 핵심 요구사항

### FR-01: 할 일 통계 카드

- **상태별 카운트** 표시: 할일 / 진행중 / 완료 / 보류
- **우선순위별 카운트**: high / medium / low
- 도넛 차트 또는 바 형태로 시각화
- 전체 할 일 수, 완료율(%) 표시
- 카드 클릭 시 할 일 탭으로 이동 (`openTab`)

### FR-02: 오늘/마감 임박 할 일 목록

- **오늘 마감** 할 일 목록 (dueDate = today, isDone = false)
- **지연된 할 일** 목록 (dueDate < today, isDone = false) — 별도 경고 표시
- **내일 마감** 할 일 목록 (dueDate = tomorrow)
- 각 항목 클릭 시 todo-detail 탭으로 이동
- 항목이 없으면 빈 상태 메시지 표시

### FR-03: 오늘 일정 카드

- 오늘 날짜의 일정 목록 (`startAt` ~ `endAt`가 오늘과 겹치는 일정)
- 시간순 정렬
- 종일 일정(allDay)은 상단에 별도 표시
- 각 항목 클릭 시 캘린더 탭으로 이동 (`searchParams: { scheduleId }`)
- 다가오는 일정(향후 7일) 미리보기 (최대 5개)

### FR-04: 최근 노트 카드

- 최근 수정된 노트 목록 (updatedAt 기준, 최대 5개)
- 노트 제목 + preview(200자) 표시
- 상대 시간 표시 (예: "2시간 전")
- 클릭 시 해당 노트 탭으로 이동
- 전체 노트 수 표시

### FR-05: 최근 캔버스 카드

- 최근 수정된 캔버스 목록 (updatedAt 기준, 최대 5개)
- 캔버스 제목 + description 표시
- 클릭 시 해당 캔버스 상세 탭으로 이동
- 전체 캔버스 수 표시

### FR-06: 파일 현황 카드

- CSV / PDF / 이미지 각각의 파일 수 표시
- 최근 추가된 파일 표시 (최대 3개씩)
- 아이콘 + 파일명으로 표시
- 클릭 시 해당 파일 탭으로 이동

### FR-07: 빠른 액션 영역

- 새 할 일 만들기 → `CreateTodoDialog` 열기
- 새 일정 만들기 → `ScheduleFormDialog` 열기
- 파일 탐색기 열기 → folder 탭 열기
- 캘린더 열기 → calendar 탭 열기

## 기술적 결정사항

### 레이아웃: 반응형 그리드 (Container Query)

TabContainer 내부에서 `@container` 기반 반응형 레이아웃 적용:

```
@[800px] 이상: 3열 그리드
@[400px] 이상: 2열 그리드
400px 미만: 1열 스택
```

### FSD 계층 구조

```
pages/dashboard/
  ui/DashboardPage.tsx          ← 기존 stub 수정 (위젯 조합)

widgets/dashboard/
  ui/TodoStatsCard.tsx           ← FR-01 할 일 통계
  ui/UpcomingTodoCard.tsx        ← FR-02 마감 임박 할 일
  ui/TodayScheduleCard.tsx       ← FR-03 오늘 일정
  ui/RecentNotesCard.tsx         ← FR-04 최근 노트
  ui/RecentCanvasCard.tsx        ← FR-05 최근 캔버스
  ui/FileOverviewCard.tsx        ← FR-06 파일 현황
  ui/QuickActionsCard.tsx        ← FR-07 빠른 액션
  index.ts                       ← barrel export

shared/ui/
  dashboard-card.tsx             ← 대시보드 카드 래퍼 (Card + 헤더 + 아이콘 + 더보기)
```

### 위젯 → 엔티티 의존성 (FSD import 규칙 준수)

```
widgets/dashboard/ → entities/{todo,schedule,note,canvas}/ (hooks 재사용)
widgets/dashboard/ → shared/ui/ (Card, Badge 등)
pages/dashboard/   → widgets/dashboard/ (조합만)
```

- widgets에서 features 레이어 import 금지 (FSD 규칙)
- 빠른 액션(FR-07)은 features 레이어의 다이얼로그를 열어야 하므로, **pages 레이어에서 callback을 위젯에 주입**하는 방식 사용

### 데이터 패칭 전략

- 각 위젯이 독립적으로 React Query 훅 사용 (데이터 패칭 분산)
- 대시보드 진입 시 모든 쿼리가 병렬 실행 (React Query 자동)
- `staleTime` 설정으로 탭 전환 시 불필요한 re-fetch 방지
- workspaceId가 없으면 빈 상태 표시 (쿼리 skip)

### 추가 백엔드 작업: 없음

모든 데이터를 기존 API로 조회 가능. 통계 계산(카운트, 필터)은 **프론트엔드에서** 수행.

## 예외 케이스 및 엣지 케이스

### EC-01: 워크스페이스 미선택

- 워크스페이스가 선택되지 않은 경우 모든 위젯에 빈 상태 표시
- "워크스페이스를 선택해주세요" 안내 메시지

### EC-02: 데이터 없음

- 할 일 0개, 일정 0개 등 각 위젯별 빈 상태 처리
- 빈 상태에서 해당 기능으로 이동할 수 있는 CTA 버튼 제공

### EC-03: 대량 데이터

- 할 일이 수백 개인 경우 통계 계산은 빠름 (메모리 내 카운트)
- 노트/캔버스/파일 목록은 최대 5개만 표시하므로 슬라이싱 처리
- 추가 최적화 불필요 (SQLite 로컬 DB, 단일 사용자)

### EC-04: 날짜 경계 처리

- "오늘" 판정: 로컬 타임존 기준 `startOfDay()` ~ `endOfDay()`
- 종일 일정(allDay): startAt 날짜만 비교
- 시간 일정: startAt ~ endAt 범위가 오늘과 겹치는지 확인

### EC-05: 실시간 반영

- 다른 탭에서 할 일 완료/일정 추가 후 대시보드 탭으로 돌아오면 데이터 반영 필요
- React Query의 `refetchOnWindowFocus` 또는 탭 활성화 시 `invalidateQueries`로 해결

## 구현 범위

### Phase 1: 공통 UI 컴포넌트

1. `shared/ui/dashboard-card.tsx` — 카드 래퍼 컴포넌트 (아이콘, 제목, 더보기 링크)

### Phase 2: 위젯 구현

1. `TodoStatsCard` — 할 일 상태/우선순위 통계 (FR-01)
2. `UpcomingTodoCard` — 마감 임박/지연 할 일 목록 (FR-02)
3. `TodayScheduleCard` — 오늘 일정 목록 (FR-03)
4. `RecentNotesCard` — 최근 노트 목록 (FR-04)
5. `RecentCanvasCard` — 최근 캔버스 목록 (FR-05)
6. `FileOverviewCard` — 파일 현황 (FR-06)
7. `QuickActionsCard` — 빠른 액션 버튼 (FR-07)

### Phase 3: 페이지 조합

1. `DashboardPage` 수정 — 위젯 배치 + 반응형 그리드
2. 빠른 액션 콜백 연결 (다이얼로그 열기)

## 비기능 요구사항

- 대시보드 로딩 시 각 위젯이 독립적으로 로딩 (Skeleton 표시)
- Container Query 기반 반응형 (viewport query 사용 금지)
- 모든 클릭 가능 항목에 hover 효과
- 다크/라이트 테마 호환 (CSS 변수 활용)

## 위험 요소

| 위험 | 영향도 | 대응 |
|------|--------|------|
| 많은 쿼리 동시 실행 → 초기 로딩 지연 | 중 | Skeleton UI + React Query 병렬 실행 |
| 위젯 개수 많아 스크롤 필요 | 하 | 우선순위 높은 위젯 상단 배치 |
| 프론트엔드 통계 계산 비용 | 하 | 단순 카운트/필터, useMemo 활용 |

## 성공 기준

- 대시보드에서 워크스페이스 전체 현황을 한눈에 파악 가능
- 각 위젯에서 해당 기능으로 원클릭 이동
- 데이터 변경 후 대시보드 재방문 시 최신 데이터 반영
- Container Query 기반 반응형 레이아웃 정상 동작
- FSD 계층 규칙 준수 (widgets → entities → shared)
